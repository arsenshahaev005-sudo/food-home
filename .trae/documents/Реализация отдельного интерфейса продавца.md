# План реализации кабинета продавца

Для того чтобы полностью заменить навигационную панель (навбар) для продавца и сохранить удобную навигацию для покупателей, я предлагаю разделить приложение на две логические части.

## 1. Реструктуризация маршрутов (Route Groups)
Мы используем возможности Next.js для создания групп маршрутов. Это позволит нам назначить разные макеты (Layouts) для разных частей сайта без изменения URL-адресов API.

*   **`src/app/(main)`** — Группа для покупателей.
    *   Сюда будут перемещены все текущие страницы: Главная, Каталог, Корзина, Профиль пользователя и т.д.
    *   У этой группы будет свой `layout.tsx` с **обычным навбаром** (Логотип, Поиск, Корзина).
*   **`src/app/(seller)`** — Новая группа для продавцов.
    *   Здесь будет находиться кабинет продавца.
    *   У этой группы будет отдельный `layout.tsx` с **новым навбаром продавца**.

## 2. Создание нового навбара (`SellerHeader`)
Я создам компонент `src/components/SellerHeader.tsx`, который будет содержать верхнее меню именно так, как вы указали:
*   **Меню**: Профиль, Заказы, Товары, Прогресс, Статистика, Финансы, Чат, Отзывы.
*   **Кнопка справа**: "Личный профиль" (для возврата в режим покупателя).

## 3. Страница кабинета (`/seller`)
*   Страница будет доступна по адресу `/seller`.
*   Внутри раздела "Профиль" (который будет открываться по умолчанию) я реализую под-меню:
    *   О магазине
    *   Условия доставки
    *   Сотрудники
    *   Реквизиты
    *   Документы
    *   Уведомления ФНС
*   Также добавлю ссылку "Ваша страница на Food&Home" в видном месте.

## 4. Обновление переходов
*   В обычном профиле (`/profile`) кнопка "Кабинет продавца" будет перенаправлять на `/seller`.
*   В кабинете продавца (`/seller`) кнопка "Личный профиль" будет возвращать на `/profile`.

Этот подход является наиболее правильным с точки зрения архитектуры и позволит в будущем легко расширять функционал продавца, не ломая интерфейс покупателя.
